# Деплой и запуск (Telegram Business)

Всё, что нужно, чтобы поднять бота в бою: что запросить у владельца, куда деплоить, пошагово.

---

## 1. Что запросить у владельца аккаунта (гадалки)

Business-режим **не даёт нам доступа к её паролю/переписке** — только токен нашего бота и разрешение отвечать. Отзывается в 1 клик.

**От неё нужно (по шагам):**
1. **Telegram Premium** — активен на её аккаунте (Business — это функция Premium). Если нет — оформить (~$5/мес) или подарить Premium на её аккаунт.
2. Включить бизнес-режим: **Настройки → Telegram для бизнеса (Telegram Business)**.
3. В разделе **«Чат-боты» (Chatbots)** добавить нашего бота **@наш_бот** и выдать права: **отвечать на сообщения** (и «управлять сообщениями» / отмечать прочитанным).
4. Прислать её **@username** (чтобы прогнать тест как «клиент»).
5. Сказать её **часовой пояс** (если захотим «тихие часы»).

Ни телефон, ни код, ни 2FA-пароль, ни api-ключи — **НЕ нужны**.

**Что делаем МЫ (не требует владельца):**
1. Создаём бота в **@BotFather** → получаем **токен**.
2. Включаем боту **Business Mode**: `@BotFather → /mybots → <бот> → Bot Settings → Business Mode → Turn On` (обязательно, иначе бот нельзя подключить к бизнес-аккаунту).
3. Даём владельцу **@username** этого бота (для шага 3 выше).
4. Деплоим (ниже) и задаём `BOT_TOKEN` + `OPERATOR_CHAT_ID`.

**Инструкция для пересылки владельцу:** *«1) Проверь, что активен Telegram Premium. 2) Настройки → Telegram для бизнеса. 3) Раздел „Чат-боты“ → добавь @наш_бот и разреши отвечать на сообщения. 4) Скинь свой @username.»*

## 2. Куда деплоить

Боту нужен **всегда включённый сервер** (телефон/домашний ПК не годятся — засыпают, рвётся связь). Логика (таймеры, БД, картинки) крутится у нас; Telegram только доставляет сообщения.

**Вариант A — VPS (рекомендую, «как обычно делают»):** Hetzner CAX11 (~€5/мес) или CX22. Домен/HTTPS **не нужны** — бот работает через long-polling. Ставим Python + зависимости, запускаем под `systemd Restart=always`.

**Вариант B — Docker (на любом хостинге):** тот же контейнер поднимается на Railway / Render / Fly.io / любом VPS. Файлы `deploy/Dockerfile` + `deploy/docker-compose.yml` готовы. Важно: `data/` и `media/` — как volume (чтобы состояние и картинки переживали рестарт), **один экземпляр** (не масштабировать репликами — одна сессия getUpdates).

> Не подойдёт: «серверлесс» с засыпанием (обычные free-tier функции), т.к. нужен постоянный процесс с таймерами и long-polling.

## 3. Пошагово на VPS (Ubuntu 24.04)

```bash
# 1. система + Python 3.12 (НЕ 3.14 — aiogram может его ещё не поддерживать)
sudo apt update && sudo apt install -y python3.12 python3.12-venv git
sudo useradd -r -m -d /opt/tarot-bot tarot

# 2. код (залить папку проекта в /opt/tarot-bot, включая «Воронка таро.xlsx»)
sudo -u tarot bash -c '
  cd /opt/tarot-bot
  python3.12 -m venv .venv
  .venv/bin/pip install -r requirements.txt
  cp .env.example .env            # затем впишите BOT_TOKEN и OPERATOR_CHAT_ID
'
# отредактируйте /opt/tarot-bot/.env  (chmod 600 автоматически задайте)
sudo chmod 600 /opt/tarot-bot/.env

# 3. импорт контента + проверки (один раз)
sudo -u tarot /opt/tarot-bot/.venv/bin/python /opt/tarot-bot/run_import.py
sudo -u tarot /opt/tarot-bot/.venv/bin/python /opt/tarot-bot/run_tests.py
sudo -u tarot /opt/tarot-bot/.venv/bin/python /opt/tarot-bot/run_qa.py

# 4. сервис 24/7
sudo cp /opt/tarot-bot/deploy/tarot-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now tarot-bot
sudo systemctl status tarot-bot
journalctl -u tarot-bot -f          # логи
```

Docker-альтернатива: заполнить `.env`, затем `docker compose -f deploy/docker-compose.yml up -d --build`.

## 4. Порядок запуска в бою (чтобы ничего не потерять)

1. Владелец подключил бота (раздел 1).
2. `BOT_TOKEN` + `OPERATOR_CHAT_ID` в `.env`, сервис поднят.
3. **Шаг 0 (спайк):** с тестового телеграм-аккаунта написать боту-владельцу кодовое слово **ТАРО** и убедиться, что через 7 мин приходит приветствие «от гадалки», а дальше вся цепочка с картинкой и разбором. Это единственная часть, которую нельзя проверить без реального аккаунта.
4. Плавный ramp-up: сначала несколько живых клиентов под присмотром, следим за `@SpamBot` и логами.

## 5. Эксплуатация

- **Бэкапы:** ежедневно копировать `data/` (SQLite) и `media/`. Для SQLite в WAL — `sqlite3 data/tarot.sqlite ".backup backup.sqlite"` (не голое `cp`).
- **Мониторинг:** `journalctl -u tarot-bot`; алерты о горячих лидах приходят в `OPERATOR_CHAT_ID`.
- **Обновление контента:** отредактировать `Воронка таро.xlsx` → `run_import.py` → перезапустить сервис.
- **Один процесс:** никогда не запускать два экземпляра с одним токеном.
- **Тихие часы / джиттер:** по умолчанию выключены (строгие тайминги). Включить джиттер — `TAROT_JITTER=1`.
