# Таро-автоответчик (Telegram Business)

Автоответчик, который отвечает **от лица гадалки** (Telegram Business, клиент не видит бота)
и ведёт входящих **строго по скрипту воронки** из `Воронка таро.xlsx`.

## Как работает (строго по воронке)
Клиент пишет кодовое слово **ТАРО** →
- **+7 мин** приветствие → сразу просьба назвать имя+запрос
- **+5 мин** «я в работе»
- **+15 мин** интро (с именем клиента) → **карта (фото)** → **разбор под ней** (текст)
- **+30 сек** CTA

66 связок раздаются **случайно и поровну** (перемешанный мешок). Каждый шаг — относительно предыдущего.

## Запуск локально (без токена — режим симуляции)
```bash
python3 -m pip install -r requirements.txt      # нужен openpyxl (aiogram — только для live)
python3 run_import.py       # разобрать xlsx -> data/tarot.sqlite + media/ (66 связок, 30 картинок)
python3 run_tests.py        # самопроверки (последовательность/тайминги/картинка/равномерность/идемпотентность)
python3 run_simulate.py     # полный прогон воронки на виртуальном времени (транскрипт)
```

## Запуск «вживую» (когда будут токен и Premium-аккаунт)
1. @BotFather → создать бота → **Bot Settings → Business Mode → Enable**.
2. Гадалка: **Telegram Premium → Настройки → Telegram для бизнеса → Чат-боты → добавить @нашего_бота** (права: отвечать/читать).
3. На сервере:
   ```bash
   export BOT_TOKEN=123:ABC
   export OPERATOR_CHAT_ID=<ваш telegram id>   # уведомления о горячих лидах
   python3 -m bot.app
   ```
   Работает через long-polling (домен не нужен). `systemd Restart=always` — для 24/7.

Пока `BOT_TOKEN` не задан — live-режим выключен (заглушка VPS/API), используйте симуляцию.

## Структура
```
bot/config.py     настройки, тайминги, кодовое слово, стоп/intent-слова
bot/importer.py   xlsx -> БД + картинки (карты по имени, картинки по якорю, ИМЯ только в r6)
bot/db.py         схема SQLite (WAL), транзакции
bot/funnel.py     роутер входящих + машина состояний (старт/ре-триггер/захват имени/стоп/hand-off)
bot/scheduler.py  планировщик (outbox: claim->send->confirm, атомарная цепочка, TTL, окно 24ч)
bot/variants.py   выбор 1 из 66 случайно-поровну
bot/content.py    карты, извлечение имени, подстановка, матчинг кодового слова
bot/transport.py           интерфейс + симулятор (для локального прогона)
bot/transport_business.py  живой aiogram-транспорт (от лица аккаунта)
bot/app.py                 live entrypoint (long-polling, business-апдейты)
bot/simulate.py            демо на виртуальном времени
```

## Заметки
- Диагностика длиннее лимита подписи (1024), поэтому: **карта отдельным сообщением → разбор отдельным** (текст под картинкой). Влезает в 4096.
- Тайминги строгие (джиттер выключен: `TAROT_JITTER=1` включит разброс для анти-бана позже).
- `data/tarot.sqlite` — состояние; `media/` — карты (истина на диске); бэкапить оба.
